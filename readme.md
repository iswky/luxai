# LuxAI - Система парсинга и анализа тендеров

Этот проект предназначен для автоматического сбора данных о тендерах с госзакупок, извлечения технических характеристик с помощью ИИ (DeepSeek) и удобного отображения результатов через веб-интерфейс.

## 🚀 Быстрый запуск через Docker

Самый простой способ запустить всё — использовать Docker Compose.

### 📋 Предварительные требования
- Docker и Docker Compose
- LLM модель (опционально, для локального использования Ollama)

### 🛠 Запуск
1. Склонируйте репозиторий.
2. Поместите ваши `.gguf` модели (рекомендуется `gemma`) в директорию `llm-service/models/` и переименуйте нужную в `model.gguf`.
3. Выполните команду:
   ```bash
   docker compose up --build
   ```

После запуска:
- **Веб-интерфейс:** [http://localhost:8000](http://localhost:8000)
- **База данных (Postgres):** `localhost:5432` (user: `user`, password: `pass`, db: `appdb`)
- **Ollama API:** [http://localhost:11434](http://localhost:11434)

---

## 🏗 Структура проекта

- `backend/` — Django проект с веб-интерфейсом.
- `parser/` — Скрипты для сбора данных с zakupki.gov.ru и парсинга PDF.
- `db/` — SQL скрипты для инициализации базы данных.
- `init/` — Скрипты инициализации для Docker контейнеров.
- `llm-service/` — Хранилище для локальных моделей ИИ.

---

## ⚙️ Компоненты системы

### 1. Веб-интерфейс (Backend)
Отображает список тендеров, их детали и позволяет управлять процессом анализа.
Основные страницы:
- `/applications/` — Список всех найденных тендеров.
- `/files/` — Управление документами.
- `/equipment/` — Каталог оборудования.

### 2. Парсер (Parser)
Находится в контейнере `parser`. Для запуска парсинга используйте:
```bash
docker exec -it parser python parser/main.py
```
- `zakupki_parser.py` — Сбор базовой информации о тендерах.
- `llm.py` — Извлечение характеристик из PDF через DeepSeek API.
- `db.py` — Сохранение результатов в базу данных.

### 3. База данных
Используется PostgreSQL 15. Схема данных автоматически создается при первом запуске контейнера с помощью `db/init_schema.sql`.

## 🛠 Ручной запуск (для разработки)

Если вы хотите запустить проект локально без Docker:

1. **База данных:**
   Запустите скрипт для настройки локальной БД:
   ```bash
   chmod +x ./setup_local_db.sh
   sudo ./setup_local_db.sh
   ```

2. **Зависимости:**
   ```bash
   pip install -r requirements.txt
   playwright install chromium
   ```

3. **Запуск Django:**
   ```bash
   python backend/manage.py runserver
   ```

4. **Запуск парсера:**
   ```bash
   python parser/main.py
   ```

## 📝 Настройка LLM
Для работы парсинга характеристик через DeepSeek необходимо установить переменную окружения:
`DEEPSEEK_API_KEY=ваш_ключ`

Либо использовать локальную Ollama, настроив её в `parser/llm.py`.
